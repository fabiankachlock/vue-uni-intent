# API Reference

Everything is imported from `vue-uni-intent`.

## `createUniIntent(options)`

Creates the Vue plugin. Install it with `app.use(...)`.

| Option         | Type                    | Default   | Description                                                        |
| -------------- | ----------------------- | --------- | ----------------------------------------------------------------- |
| `adapters`     | `InputAdapter[]`        | —         | Input sources to install. Required and explicit.                  |
| `wrap`         | `boolean`               | `false`   | Wrap around to the opposite edge when navigating past the last trigger. |
| `initialFocus` | `"first" \| "none"`     | `"first"` | Initial focus strategy for the root layer.                        |

## `useTrigger(options)`

Registers a controllable trigger.

**Options**

| Option      | Type                          | Description                                                      |
| ----------- | ----------------------------- | --------------------------------------------------------------- |
| `id`        | `string`                      | Unique within its layer.                                        |
| `onTrigger` | `(cause: TriggerCause) => void` | Called on activation or a matching shortcut; receives what fired it (see [Trigger cause](#trigger-cause)). |
| `onFocus`   | `(cause: FocusCause) => void` | Optional. Called whenever this trigger gains focus; receives what caused it (see [Focus cause](#focus-cause)). |
| `shortcuts` | `ShortcutDescriptor[]`        | Inputs that fire this trigger regardless of focus.              |
| `disabled`  | `MaybeRefOrGetter<boolean>`   | Disabled triggers are skipped by navigation and shortcuts.      |
| `autofocus` | `boolean`                     | Receive the layer's initial focus.                              |

**Returns**

| Field     | Type                        | Description                                            |
| --------- | --------------------------- | ------------------------------------------------------ |
| `ref`     | `Ref<HTMLElement \| null>`  | Bind to the element. Leave unbound for a pure shortcut. |
| `focused` | `ComputedRef<boolean>`      | Whether this trigger is currently focused.             |
| `focus`   | `() => void`                | Programmatically focus this trigger.                   |
| `trigger` | `() => void`                | Programmatically activate this trigger (fires `onTrigger` with a `"manual"` cause). |

## Trigger cause

`onTrigger` receives a `TriggerCause` describing what fired it. The core is
input-agnostic, so the shape is open — every cause carries a `source` and a
`via`, and each adapter attaches its own detail:

| Field    | Type                                     | Description                                                              |
| -------- | ---------------------------------------- | ------------------------------------------------------------------------ |
| `source` | `string`                                 | The firing adapter's `name`, or `"manual"` for a `.trigger()` call.      |
| `via`    | `"activate" \| "shortcut" \| "manual"`   | Direct activation, a shortcut match, or a programmatic call.             |
| `event`  | `Event \| undefined`                     | The native DOM event behind the input, when the source has one.          |
| …        | `unknown`                                | Adapter-specific detail (e.g. the gamepad `button` index).               |

Built-in adapters export a precise cause type and a guard that narrows to it:

| Guard             | Narrows to            | Extra fields                       |
| ----------------- | --------------------- | ---------------------------------- |
| `isKeyboardCause` | `KeyboardTriggerCause`| `event: KeyboardEvent`             |
| `isMouseCause`    | `MouseTriggerCause`   | `event: MouseEvent`                |
| `isGamepadCause`  | `GamepadTriggerCause` | `button: number` (no native event) |
| `isManualCause`   | `ManualTriggerCause`  | —                                  |

## Focus cause

`onFocus` receives a `FocusCause`, the focus-side counterpart of `TriggerCause`.
It fires on **every** focus change and describes how focus was gained:

| Field       | Type                    | Description                                                            |
| ----------- | ----------------------- | --------------------------------------------------------------------- |
| `source`    | `string`                | The adapter's `name`, `"manual"` for a `focus()` call, or `"core"`.   |
| `via`       | `FocusVia`              | How focus was gained (see below).                                     |
| `direction` | `Direction \| undefined`| Set when `via` is `"navigate"`.                                       |
| `event`     | `Event \| undefined`    | The native DOM event behind the input, when the source has one.       |
| …           | `unknown`               | Adapter-specific detail.                                              |

`via` is one of:

| `via`           | When                                                                    |
| --------------- | ----------------------------------------------------------------------- |
| `"navigate"`    | Spatial navigation (an adapter's `move`); `direction` is set.           |
| `"focus"`       | Direct focus of a trigger (mouse hover/click, an adapter's `focus(id)`).|
| `"programmatic"`| A consumer-called `useTrigger().focus()`.                               |
| `"tab"`         | Native focus change adopted via `focusin` (e.g. Tab).                   |
| `"restore"`     | A layer became active again and its remembered focus was restored.      |
| `"initial"`     | A layer/app resolved its initial focus.                                 |
| `"cleanup"`     | The focused trigger was removed and focus fell back to a survivor.      |

Built-in adapters export a precise cause type and a guard:

| Guard                  | Narrows to          | Extra fields                              |
| ---------------------- | ------------------- | ----------------------------------------- |
| `isKeyboardFocusCause` | `KeyboardFocusCause`| `via: "navigate"`, `direction`, `event: KeyboardEvent` |
| `isMouseFocusCause`    | `MouseFocusCause`   | `via: "focus"`, `event: MouseEvent`       |
| `isGamepadFocusCause`  | `GamepadFocusCause` | `via: "navigate"`, `direction` (no native event) |
| `isManualFocusCause`   | `ManualFocusCause`  | `via: "programmatic"`                     |
| `isCoreFocusCause`     | `CoreFocusCause`    | core-resolved focus (`tab` / `restore` / `initial` / `cleanup`) |

## `useTriggerLayer(options?)`

Scopes descendant triggers to a new focus layer for the component's lifetime.

- **Options:** `id?` (auto-generated if omitted), `initialFocus?` (a trigger id
  to focus when the layer activates).
- **Returns:** `{ id, isActive }` — `isActive` is a `ComputedRef<boolean>` that
  is `true` while the layer is topmost.

`ROOT_LAYER_ID` is exported for referencing the always-present base layer.

## Adapters

- **`keyboardAdapter(options?)`** — `options.keys` maps `up` / `down` / `left` /
  `right` / `activate` to arrays of `KeyboardEvent.key` values.
- **`mouseAdapter()`** — hover to focus, left-click to activate; non-left
  buttons dispatch shortcuts.
- **`gamepadAdapter(options?)`** — `deadzone`, `initialRepeatDelay`,
  `repeatInterval`. Also exports `pollGamepads`, `createGamepadPollState` for
  custom polling loops.

Adapters fire triggers via `ctx.activate(cause)` and
`ctx.dispatchShortcut(input, cause)`, passing a [`TriggerCause`](#trigger-cause).
They move focus via `ctx.move(direction, cause)` and `ctx.focus(id, cause)`,
passing an optional [`FocusCause`](#focus-cause) that reaches the newly focused
trigger's `onFocus`.

## Shortcut helpers

```ts
import { key, button, mouseButton } from "vue-uni-intent";

key("Escape"); // { key: "Escape" }
key("s", { ctrl: true }); // { key: "s", ctrl: true }
button("B"); // { button: 1 } — standard-mapping index via GamepadButton
mouseButton("Back"); // { mouseButton: 3 }
mouseButton("Right", { ctrl: true }); // { mouseButton: 2, ctrl: true }
```

Name→index constants are exported for readability and typo-safety:
`Key`, `GamepadButton`, `GamepadAxis`, and `MouseButton`
(with `resolveGamepadButton` / `resolveMouseButton`).

## Low-level exports

For custom adapters or testing outside Vue:

- **`findNext(from, candidates, direction, options?)`** — the pure spatial
  navigation function.
- **`matchShortcut(input, descriptor)`** — match a raw input against one
  shortcut descriptor.

All public types (`UniIntentOptions`, `UseTriggerOptions`, `InputAdapter`,
`AdapterContext`, `ShortcutDescriptor`, …) are exported as well.
