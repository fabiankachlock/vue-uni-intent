# vue-uni-intent

Game-menu-style button control for Vue 3: **one focused trigger at a time**,
navigable with keyboard, mouse, and gamepad through a single API.

- **Automatic spatial navigation.** "Where does _up_ go?" is answered from the
  DOM positions of the registered triggers — there is no focus graph to
  maintain.
- **Explicit, swappable adapters.** Keyboard, mouse, and gamepad ship as
  separate input adapters that you configure by hand. The core only knows the
  `InputAdapter` contract, so input sources can be omitted, replaced, or added.
- **Unified shortcuts.** `Escape`, gamepad `B`, and the mouse back button can
  all fire the same trigger, regardless of what is focused.
- **Focus layers.** A modal can confine navigation and shortcuts to its own
  triggers, then restore the previous focus when it closes.
- **Real, accessible focus.** Triggers use `element.focus()`, and every trigger of
  the active layer stays in the native Tab order, so <kbd>Tab</kbd> moves between
  triggers and each landing is adopted as focus. A `data-uni-focused` attribute is
  exposed for styling.
